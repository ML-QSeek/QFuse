## 这是什么

**QFuse 是一个以主题为核心的 AI 驱动 HTML 应用框架。**

它让 AI 围绕一个主题，持续生成可交互、可操作的 HTML 应用页面。这些页面覆盖了从讨论、分析、路演到报告输出的完整工作流。

**它不是一个聊天机器人——它是一个以主题为纽带，让 AI 和人在同一个界面里协作完成知识工作的运行环境。**


## 核心理念

| 理念 | 含义 |
|------|------|
| **主题为核心** | 每一个主题都是一个独立的知识单元，汇聚了讨论、数据、图表、报告等所有产出 |
| **AI 生成应用** | AI 的产出不是文本，而是完整的 HTML 应用页面 |
| **浏览器即终端** | 人只需浏览器，即可运行 AI 交付的任何应用界面 |
| **页面即资产** | 每一个页面都可保存、版本化、追溯，成为可复用的知识资产 |


## 应用场景

### 主题研讨
围绕一个主题，AI 角色与人类协作完成信息收集、分析讨论、观点碰撞，所有过程以 HTML 页面实时呈现。

### 数据报告
AI 根据主题自动生成结构化报告，包含数据可视化、分析结论、建议方案，以完整的 HTML 报告页面交付。

### 路演展示
AI 将主题内容生成为路演页面，可直接投屏展示，包含标题、结构、图表、核心观点，无需二次整理。

### 晨会
AI 自动聚合隔夜信息、市场动态、持仓分析、今日要点，生成晨会 HTML 页面，团队在浏览器中共同浏览和讨论。


## 核心能力

### 主题驱动

每一个工作单元都是一个“主题”：

- 创建主题，设定标题
- 围绕主题持续生成和迭代页面
- 所有产出归属于主题，可追溯、可版本化

### 页面生成

AI 根据主题上下文，动态生成各类 HTML 应用页面：

- 路演页面（PPT 风格，可直接投屏）
- 数据看板与可视化面板
- 结构化报告与分析文档
- 晨会页面（信息聚合+今日要点）

### 沉淀与复用

所有页面均可沉淀：

- 覆盖保存：更新当前版本
- 创建保存：新建分支版本
- 提炼保存：萃取精华生成新页面


## 交互方式

QFuse 以 **HTML 为交互核心**：

- AI 围绕主题实时生成 HTML 页面
- 人通过浏览、点击、填写、编辑与系统协作
- 同一个主题下的所有页面构成完整的工作流
- 会话只是辅助手段，界面才是核心交互形态


## 与 QSeek 的关系

QFuse 与 QSeek 共同构成 Molin 的核心驱动：

- **QSeek** 负责推理和判断
- **QFuse** 将推理结果生成为可运行的 HTML 应用
- 人通过 QFuse 页面与 QSeek 的推理结果协作
